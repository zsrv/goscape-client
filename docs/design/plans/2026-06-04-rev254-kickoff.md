The class-level `@ObfuscatedName("X")` annotation is the line immediately preceding `public (final )?(class|interface)`. Member keys contain a dot (`"pb.d"`); class keys do not — filter on that.

```bash
cd $HOME/Code/github.com/LostCityRS/Client-Java
for pin in 176a85f 2e62978; do
  out=/tmp/claude-1000/obfmap-$pin.tsv; : > "$out"

export const meta = {
  name: 'scope-254-delta',
  description: 'Distill verified 245.2→254 Java logic delta (method-paired, adversarially verified)',
  phases: [
    { title: 'Pair', detail: 'Client.java method name-set pairing' },
    { title: 'Compare', detail: 'per-unit base↔target comparison' },
    { title: 'Verify', detail: 'adversarial re-check of claimed real deltas' },
    { title: 'Opcodes', detail: 'full opcode table re-derivation + multiset diff' },
  ],
}
const { repo, base, target, units } = args
const READ = `Read Java sources ONLY via: git -C ${repo} show <pin>:<path>  (pins: base=${base}, target=${target}). Never read working trees.`
const CHURN = `Known churn classes to EXCLUDE from real deltas: locals/params renamed (incl. regression to argN), member-level @ObfuscatedName key reassignment, blank-line/comment reflow, commutative operand reorder, final-drops. Class-name changes are already mapped (World→ClientBuild, World3D→World, Component→IfType, Jagfile→JagFile, io/Protocol→client/Protocol) — treat them as churn, not deltas.`
const DELTAS_SCHEMA = { type:'object', required:['unit','methodNameSetOk','deltas'], properties:{
  unit:{type:'string'}, methodNameSetOk:{type:'boolean'},
  unpaired:{type:'object',properties:{baseOnly:{type:'array',items:{type:'string'}},targetOnly:{type:'array',items:{type:'string'}}}},
  deltas:{type:'array',items:{type:'object',required:['title','kind','javaRef','description','size'],properties:{
    title:{type:'string'}, kind:{enum:['real','signature-only']},
    javaRef:{type:'string'}, description:{type:'string'}, size:{enum:['S','M','L','XL']}}}},
  churnNotes:{type:'string'}}}
const VERDICT_SCHEMA = { type:'object', required:['verdict','reason'], properties:{
  verdict:{enum:['confirmed','refuted','amended']}, reason:{type:'string'}, amendedDescription:{type:'string'}}}
const PAIR_SCHEMA = { type:'object', required:['pairs','baseOnly','targetOnly'], properties:{
  pairs:{type:'array',items:{type:'string'}}, baseOnly:{type:'array',items:{type:'string'}}, targetOnly:{type:'array',items:{type:'string'}}}}

const comparePrompt = (label, basePath, targetPath, extra) => `You are comparing one Java unit across two revisions to find the REAL game-logic delta.
${READ}
Base: ${basePath ? `${base}:${basePath}` : 'ABSENT (new class in 254)'}
Target: ${targetPath ? `${target}:${targetPath}` : 'DELETED in 254'}
${extra}
Pair methods BY NAME; first check the name sets match 1:1 (report unpaired names; attempt signature/body-shape pairing for them, never silently drop). For each paired method compare bodies semantically. ${CHURN}
For a new/deleted class, the whole class is one or more 'real' deltas describing what must be added/removed in the Go port.
Classify every difference: kind 'real' (behavioral/structural — must be ported) or 'signature-only' (type/visibility/param-order with no behavior change — port only if it affects Go signatures). Cite javaRef as File.java:NNN @${target.slice(0,7)} (or @${base.slice(0,7)} for removals). Size: S<20 changed lines, M<100, L<400, XL beyond.`

phase('Pair')
const pairing = await agent(`Extract the full method name list of jagex2/client/Client.java from BOTH pins.
${READ} Paths: base=src/main/java/jagex2/client/Client.java target=src/main/java/jagex2/client/Client.java
Return pairs = names present in both, baseOnly, targetOnly. Names only (no signatures).`,
  { schema: PAIR_SCHEMA, label: 'pair:Client' })
log(`Client.java: ${pairing.pairs.length} paired, ${pairing.baseOnly.length} base-only, ${pairing.targetOnly.length} target-only`)
const chunks = []
for (let i = 0; i < pairing.pairs.length; i += 20) chunks.push(pairing.pairs.slice(i, i + 20))
const clientItems = chunks.map((c, i) => ({ name: `Client[${i}]`,
  basePath: 'src/main/java/jagex2/client/Client.java', targetPath: 'src/main/java/jagex2/client/Client.java',
  extra: `ONLY compare these methods (paired by name): ${c.join(', ')}` }))
if (pairing.baseOnly.length || pairing.targetOnly.length) clientItems.push({ name: 'Client[unpaired]',
  basePath: 'src/main/java/jagex2/client/Client.java', targetPath: 'src/main/java/jagex2/client/Client.java',
  extra: `ONLY analyze these UNPAIRED methods. base-only: ${pairing.baseOnly.join(', ') || '(none)'}; target-only: ${pairing.targetOnly.join(', ') || '(none)'}. Try body-shape pairing across the lists (renamed methods); whatever truly has no counterpart is a real add/remove delta.` })

phase('Compare')
const results = await pipeline(
  [...units.map(u => ({ ...u, extra: '' })), ...clientItems],
  item => agent(comparePrompt(item.name, item.basePath, item.targetPath, item.extra),
    { schema: DELTAS_SCHEMA, phase: 'Compare', label: `cmp:${item.name}` }),
  (res, item) => !res ? null : parallel(res.deltas.filter(d => d.kind === 'real').map(d => () =>
      agent(`Adversarially verify this claimed 245.2→254 Java delta. Try to REFUTE it as churn (renames/reorder/reflow). ${READ}
Claim: ${d.title} — ${d.description} (ref ${d.javaRef}; unit base=${item.basePath} target=${item.targetPath})
Verdict 'refuted' if it is churn or misread; 'amended' (with amendedDescription) if real but mischaracterized; else 'confirmed'.`,
        { schema: VERDICT_SCHEMA, phase: 'Verify', label: `vfy:${item.name}:${d.title.slice(0, 30)}` })
        .then(v => ({ ...d, verdict: v }))))
      .then(verified => ({ ...res, deltas: [...verified.filter(Boolean), ...res.deltas.filter(d => d.kind !== 'real')] }))
)

phase('Opcodes')
const OPTABLE_SCHEMA = { type:'object', required:['tables'], properties:{ tables:{type:'object',properties:{
  outbound:{type:'array',items:{type:'object',required:['opcode','label'],properties:{opcode:{type:'integer'},label:{type:'string'},lengthInfo:{type:'string'}}}},
  inbound:{type:'array',items:{type:'object',required:['opcode','label'],properties:{opcode:{type:'integer'},label:{type:'string'},lengthInfo:{type:'string'}}}},
  zone:{type:'array',items:{type:'object',required:['opcode','label'],properties:{opcode:{type:'integer'},label:{type:'string'},lengthInfo:{type:'string'}}}},
  serverprotSizes:{type:'array',items:{type:'integer'}}}}, notes:{type:'string'}}}
const opPrompt = pin => `Exhaustively re-derive ALL wire opcode tables from the ${pin === base ? '245.2 base' : '254 target'} Java client at pin ${pin}.
${READ}
Sources: jagex2/client/Client.java (outbound writes — pisaac/p1 opcode calls with deob label comments; inbound packetType dispatch; zone-update dispatch) and the Protocol class (${pin === base ? 'jagex2/io/Protocol.java' : 'jagex2/client/Protocol.java'} — SERVERPROT length table).
Enumerate EVERY opcode: never sample, never carry values from another revision. Include the deob // LABEL comment (or dispatch-site description) per opcode.`
const [opsBase, opsTarget] = await parallel([() => agent(opPrompt(base), { schema: OPTABLE_SCHEMA, label: 'ops:base' }),
                                             () => agent(opPrompt(target), { schema: OPTABLE_SCHEMA, label: 'ops:target' })])
const opsVerify = await agent(`Two opcode-table extractions follow. Re-derive the 254 target tables YOURSELF from source and report any disagreement with extraction B; also sanity-check A against the Go port's existing 245.2 tables if helpful. ${READ}
A(base ${base.slice(0,7)}): ${JSON.stringify(opsBase?.tables ?? {})}
B(target ${target.slice(0,7)}): ${JSON.stringify(opsTarget?.tables ?? {})}`,
  { schema: { type:'object', required:['agreement','discrepancies'], properties:{ agreement:{type:'boolean'}, discrepancies:{type:'array',items:{type:'string'}}}}, phase:'Opcodes', label:'ops:verify' })

return { clientPairing: pairing, units: results.filter(Boolean), opcodes: { base: opsBase, target: opsTarget, verify: opsVerify } }
```
