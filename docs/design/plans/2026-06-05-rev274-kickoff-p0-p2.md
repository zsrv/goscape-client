### Task 4: P1 — Extract class-level obf-key tables from both pins

**Files:** Create (untracked scratch): `audit-274/classes-254.txt`, `audit-274/classes-274.txt`

- [ ] **Step 1: Extract key↔class tables**

The class-level `@ObfuscatedName("xx")` is the first annotation in each file (member keys contain a dot, class keys don't). Avoid heredocs (the `\!` escaping trap in this harness) — run as a direct loop:

```bash
mkdir -p audit-274
JREPO=$HOME/Code/github.com/LostCityRS/Client-Java
for spec in "2e62978:254" "32f3062:274"; do
  pin=${spec%:*}; rev=${spec#*:}
  git -C "$JREPO" ls-tree -r --name-only "$pin" -- src/main/java/jagex2 src/main/java/sign \
  | while read -r f; do
      key=$(git -C "$JREPO" show "$pin:$f" | grep -m1 '@ObfuscatedName("[^".]*")' | sed 's/.*("\([^"]*\)").*/\1/')
      echo "${key:-NOKEY} $(basename "$f" .java) $f"
    done | sort > "audit-274/classes-$rev.txt"
done
wc -l audit-274/classes-254.txt audit-274/classes-274.txt
```

Expected: ~73 rows per file (74 files minus `signlink.java` if it carries no class-level key; `NOKEY` rows are fine — adjudicate manually in Task 5).
